Story

Imagine that you want to create a mobile app and you want to cater your app to both Android and iOS devices. You would need to develop two separate apps, one for the Google Play Store and one for the iOS App Store. But with Flutter, you would be able to develop under one codebase and still cater to both mobile markets, which saves you a lot of time and effort. So, what is Flutter?

What is Flutter?

Flutter is a software development kit (SDK) made by Google to develop mobile apps. It is a framework written in Dart, a programming language created by Google in 2011. Flutter is a relatively new framework; its stable 1.0 version was released in December 2018. Despite being new, it has a very active community. In 2018 - 2019, Flutter was the third most popular and second fastest growing open source project on GitHub.

Why Flutter?

There are so many different options out there to consider when wanting to develop a mobile app. You could write native apps with Java/Kotlin or Swift/Objective-C, native-like apps with React Native, or hybrid apps with Ionic or PhoneGap. Why use Flutter?

Because Flutter is...

written in one codebase

Since Flutter is a cross-platform framework, its Dart-based codebase would help you build mobile apps for both iOS & Android at once. So with Flutter development, you essentially have a single app that can be used successfully on multiple platforms.

open-source and free

There is an active community that contributes to the Flutter source code on its GitHub repository, helping the dev team fix bugs and suggest improvements.

extremely fast in performance

Native apps written in Java/Kotlin for Android or Swift/Objective-C for iOS have the best performance on their respective platforms and they also have full access to the phone's instruments (e.g. camera, microphone, etc.). Flutter is a native-like framework that compiles down into code that runs at speeds close to native apps.

extremely fast to develop for

There is a process in Flutter development known as hot reload. Imagine you are working on a Flask app and you fill out a long form and you realize it doesn't work for whatever reason. You make a change in your codebase and then you reload the entire app to see the changes. This doesn't have to be the case in Flutter. Once you make changes in your app, you can simply press a button (typically "r") on your keyboard and you get to see the changes within seconds and you also keep any state your app was in (e.g. filled in forms).

beautiful

Flutter comes with pre-made widgets and components for developers to use. These widgets can either be from a Material UI (Android) or Cupertino (iOS) library. However, the developer has flexibility to create their own style of widgets because Flutter is so malleable.

Why does Flutter use Dart?

Dart was a relatively obscure language before the advent of Flutter. However, because of Flutter's popularity, Dart became the fastest growing language on GitHub during 2018 - 2019. So why does Flutter use Dart?

Because Dart is...

an Object Oriented Programming (OOP) language

Everything in Dart can be made into a class or object. This is important for Flutter because Flutter uses classes and inheritance for all of its widgets.

an Ahead of Time (AOT) compiled language

This means that Dart code is compiled into native-like machine code that runs incredibly quickly on iOS or Android devices, as if you were actually programming in Java or Swift.

a Just in Time (JIT) compiled language

This means that Dart code can be compiled as it is being executed. This is important for the quick development process with hot reload and hot restart.

similar to JavaScript

Dart is in the C family in terms of syntax, and has a lot of crossover with JavaScript and is easy to learn.

dynamically and statically typed

Dart is also a good bridge for JavaScript users because it allows for dynamic typing, but is also safer than JavaScript because it also supports static typing.

import 'dart:math';

class Point {
    int x;
    int y;

    Point(int x, int y) {
    this.x = x;
    this.y = y;
    }

    String toString() {
    return "($x, $y)";
    }

    double distance(Point other) {
    return sqrt(pow(x - other.x, 2) + pow(y - other.y, 2));
    }
}

main() {
    Point point = new Point(1,2);
    Point other = new Point(10, 4);

    print("Distance between $point and $other is ${point.distance(other)}");
}
How to Flutter

Now that we've had a basic overview of the Dart programming language, we can start to look at what Flutter has to offer.

Basic Flutter Concepts

What are Widgets?

Everything in Flutter is made up of widgets. A widget is similar to HTML elements like a div or a table. It is basically a UI component with its own set of layout rules and properties. An entire Flutter app is built by composing these widgets together in a cohesive manner. Once you enough of these widgets, you'll get what is called a widget tree.

How are Widgets Organized?

Web pages have a Document Object Model (DOM) and Flutter has a Widget Tree. The DOM and the Widget Tree are very similar to one another. Widget tree describes how widgets are composed with each other. Below is an image showing the similarities between the two.

The following animation shows a simplified widgit tree for a more concrete app. It also introduces the concept of state management. Notice that when the user clicks add next to one of the items, that item is added to the cart.

What is State?

In Flutter, there are stateless and stateful widgets. Stateless widgets are widgets that do not change once the app is created. They can only hold information upon initialization. On the other hand, stateful widgets can hold information that changes over time. In the animation above, the cart is an example of a stateful widget because it holds information about what items the user wants to purchase. This information can change over time, requiring the user to set the state of the widget.

Each StatefulWidget contains a State object which handles all changes to state. The State object holds functions that invoke a setState function.

What is the Build Function?

build is a function in every Flutter widget that is called as soon as the screen renders a widget. The build function should return a Widget (this can be a complicated tree or a simple widget). One thing to note about the build function is that it is called everytime state is set. Even though this means we may be rerendering an entire widget tree, Flutter is very optimized and this rerendering just takes a split second.

import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget { // This widget is the root of your application.

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Flutter Demo',
      home: MyHomePage(title: 'Flutter Demo Home Page'),
    );
  }
}

class MyHomePage extends StatefulWidget {

  MyHomePage({Key key, this.title}) : super(key: key);
  // This class is the configuration for the state. It holds the values (in this case the title) provided by the parent (in this
  // case the App widget) and used by the build method of the State. Fields in a Widget subclass are always marked "final".

  final String title;

  @override
  _MyHomePageState createState() => _MyHomePageState();
}

class _MyHomePageState extends State<MyHomePage> {

  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text(widget.title), // Here we take the value from the MyHomePage object and use it to set our appbar title.
      ),
      body: Center( // Center is a layout widget. It takes a single child and positions it in the middle of the parent.
        child: Column(
          // Column is also layout widget. It takes a list of children and arranges them vertically. By default, it sizes itself
          // to fit its children horizontally, and tries to be as tall as its parent. Column has various properties to control
          // how it sizes itself and how it positions its children. Here we use mainAxisAlignment to center the children
          // vertically; the main axis here is the vertical axis because Columns are vertical (the cross axis would be horizontal).
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text(
              'You have pushed the button this many times:',
            ),
            Text(
              '$_counter',
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        child: Icon(Icons.add),
      ), // This trailing comma makes auto-formatting nicer for build methods.
    );
  }
}
Structure and Development

Now we can look at some basic commands to get Flutter apps up and running, as well as the directory structure of a basic Flutter app.

How to run a Flutter app

flutter create <app_name> - instantiates the default Flutter app (demoed above) in the current directory.

flutter run - runs the app in an emualtor or deploys the app to a mobile device.

An emulator can be created using Android Studio or Xcode. Both of these are integral to developing to support platform dependencies.

A Flutter app can be deployed to an Android phone via USB once developer settings are enabled. iPhones have a more complicated system of securities but it is possible to deploy onto an iPhone as well.

Directory Structure

/android and /ios - contain files to run the app on Android and iOS devices.

/test - is where your test files go if you want to engage in Test Driven Developemnt (TDD).

/lib - is where you will put all of your Dart files. You will spend most of your time in this folder.

pubspec.yaml - is the dependency file of your app. It will contain any packages, assets, images, fonts, etc. that you use.
